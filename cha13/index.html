깃,깃허브

클론: 깃허브 저장소에 있는 파일들을 로컬 컴퓨터로 복사본을 만든다 쉽게 얘기하면 깃허브에 있는 파일,프로젝트를 내 컴퓨터로 불러올수있다

저장소에 파일을 올리는 과정 -> 깃허브에서는 변경 사항을 커밋(저지르다)이라고 한다 커밋 과정은 add -> commit -> push 순서로 명령을 수행해 이루어진다 정리하면, 내 코드를 추가(add하고) 저지르고(commit) 저장소로 민다(push)

pull(당기다) 풀은 깃허브 저장소에 있는 파일을 가져오는 작업을 말한다 코드를 내려받는 점에서 비슷한데 이 둘은 다르다 클론은 처음 코드를 다운로드할 때 사용하고, 풀은 그 이후에 사용한다 클론하면 자동으로 깃허브에 있는 main 브랜치의 변경 내용을 추적한다 그래서 풀로 변경 내용만 내려받을 수 있다 따라서 pc와 노트북 각각에 클론을 한 이후에, 노트북으로 커밋한 코드를 pc에서 불러오려면 클론이 아니라 풀을 해야한다
